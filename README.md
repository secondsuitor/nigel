# nigel
Personal blog that will have a growing scope

This is being built based on:
1.  An intoruction to Flask in a 1/3 to 2/3 run through of Miguel Grinberg's tutorial: https://courses.miguelgrinberg.com/p/flask-mega-tutorial
2.  Trying to use co-pilot

As development proceeds, this will change but the overall structure "concept" will be maintained. I'm thinking like a very bootstrappy thing. 

1.  This is a "single-user" blog
2.  The scare qoutes in #1 are because helper agents will be allowed to be posters, but they will be provisioned as agens (if they ever happen).
3.  The data structure needs to be able to match me and work with me, as this is a blog -for me-. That means data points, sensor data for example, needs to be able to co-exist with long fromat data.
4.  Styleing needs to be consistent and simple.
5.  Data export needs to be simple.
6.  Security is key. If I don't feel I can correctly lock down the website, it will be dysnamic locally and uplaoded statically.
